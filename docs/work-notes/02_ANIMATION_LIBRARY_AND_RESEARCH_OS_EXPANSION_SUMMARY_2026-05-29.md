# 动画库扩展与个人研究系统孵化总结

日期：2026-05-29

本轮工作延续上一份 worknote 之后的进展：一方面继续把 `sci-animation` 从动力学动画示例扩展为更通用的科学概念动画库，新增了蒙特卡罗估计 pi 与高尔顿板两个概率可视化示例；另一方面，讨论并初步定义了一个更大的个人研究操作系统愿景，即在专题项目之外建立基础设施层与交叉项目演进层，帮助多个研究主题既保持聚焦，又能相互反哺。

## 1. 从动力学动画走向科学概念动画

上一阶段的重点是曲柄摇杆、弹簧振子和三层剪切楼地震响应，这些示例主要服务机械与结构动力学表达。本轮新增的蒙特卡罗和高尔顿板把项目扩展到了概率与数值方法可视化。

这个变化很重要。它说明当前项目不应只被理解为“动力学动画仓库”，而更接近一个“可运行概念库”：每个主题都需要有最小模型、可视化场景、理论说明、渲染产物和可复用接口。动画不只是结果展示，而是把抽象理论转化为可观察过程的一种表达方式。

## 2. 蒙特卡罗估计 pi 动画

蒙特卡罗示例建立了一个新的概率模型模块，用随机点在正方形内采样，并根据落入单位圆的比例估计 pi。模型层生成随机样本、圆内判断和估计序列，场景层则负责展示左侧采样区域和右侧估计值收敛曲线。

这个示例的主要价值有两点。第一，它验证了当前 `TimeResponse` 和时程曲线组件可以用于非动力学场景，只要把横轴理解为样本数而不是时间。第二，它促使 `TimeHistoryPlot` 增加了横轴标签和大跨度横轴刻度的适配能力，使通用绘图组件不再只绑定物理时间。

该示例还补充了对应报告 `docs/reports/monte-carlo-method.md`，说明面积比例、大数定律和估计公式，并生成了 mp4 与 GIF 预览。渲染过程中发现低清画面里复杂 TeX 公式显示不稳定，因此场景顶部公式改用普通文字表达，保证视频读者能清楚理解。

## 3. 高尔顿板与二项分布动画

高尔顿板示例进一步展示了随机过程如何形成分布形状。模型层预生成每个小球在每一层向左或向右的选择，计算最终槽位、累计计数和理论二项分布概率。场景层用三角钉阵、小球分批下落、底部柱状堆积、二项分布曲线和正态近似曲线共同表达随机选择的累积结果。

第一版没有做真实碰撞物理仿真，而是采用预生成离散路径。这是有意选择：当前目标是解释“独立随机选择如何形成分布”，不是求解小球碰撞动力学。预生成路径也让结果可复现，便于报告和后续调参。

该示例同样补充了理论报告 `docs/reports/galton-board.md`，并生成了 `GaltonBoardScene.mp4` 与 `galton_board.gif`。抽帧检查中发现图例在低清下不够清楚，因此改为短线加标签的形式，保证 GitHub 预览和视频截图仍然可读。

## 4. 架构与文档入口的扩展

这轮工作继续沿用“模型层、场景层、报告层、媒体层”的组织方式。新增的概率模型位于 `src/sci_animation/models/`，并通过 `src/mechanisms/` 保持兼容导出。场景仍位于 `scenes/`，报告仍进入 `docs/reports/`，GIF 预览仍放在 `media/previews/`。

相关入口也同步更新：

- `README.md` 增加蒙特卡罗和高尔顿板示例。
- `c.md` 增加两个新场景的渲染命令和结果路径。
- `docs/reports/INDEX.md` 增加概率可视化报告入口。
- `docs/specs/ROADMAP.md` 标记新增概率示例。
- `docs/specs/ARCHITECTURE.md` 记录新模型和场景边界。

这使项目的使用路径保持一致：新增动画不只是增加一个脚本，还要有模型入口、渲染命令、理论报告和可预览媒体。

## 5. 从动画项目延伸出的个人研究系统

在实现概率动画之后，本轮讨论转向更高层的问题：这个项目不只是生成动画，而是在探索一种“思考机器”的形态。用户需要的不是一个不断膨胀的单仓库知识库，而是一套能与个人研究过程绑定、服务多个主题、并持续提高认知能力的工作系统。

最终形成了一个三层框架：

1. 基础设施层：负责模板、规则、索引、状态同步、回顾机制和 AI 协作规范。
2. 专题项目研究层：每个研究主题保持独立仓库，例如当前的 `sci-animation`。
3. 交叉项目演进层：从多个项目中提炼方法、概念迁移、表达模式和个人判断变化。

这个框架的价值在于同时保护“深度”和“丰富性”。专题仓库负责做深，跨项目层负责把经验转化为可复用方法，基础设施层负责让整个系统可持续运行。

## 8. 当前状态与后续事项

当前 `sci-animation` 已经包含机械、结构动力学、仿真 replay、蒙特卡罗和高尔顿板多个示例，项目语义正在从“动力学动画库”扩展为“科学概念动画与可视化表达框架”。这个扩展是积极的，但也带来新的边界问题：哪些内容属于当前动画库，哪些内容应抽象到更高层的研究基础设施。

后续建议优先处理三件事：

1. 在 `sci-animation` 中补一个 `STATE.md`，让项目能快速说明当前状态和下一步。
2. 重新核验或补建 `henri-lab`，使三层研究系统有独立承载空间。
3. 从蒙特卡罗和高尔顿板中提炼“概率类动画模板”，避免后续每个概率示例都重新组织模型、曲线、说明和报告。

总体上，本轮工作完成了两个方向的推进：动画库在内容上变得更广，研究系统在概念上变得更清楚。下一阶段的关键不是继续无边界增加示例，而是让项目状态、专题边界和跨项目沉淀机制同步成熟。
